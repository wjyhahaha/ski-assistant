# Ski Assistant 产品说明文档

## 一、产品定位

**Ski Assistant** 是一个面向滑雪爱好者的 AI 出行服务助手，覆盖从"想去滑雪"到"滑完复盘"的全链路需求。

**核心价值**：让用户用自然语言一句话完成原本需要打开多个 App/网站才能搞定的事情——选雪场、查价格、做攻略、看票价、盯预售、分析滑雪姿态。

---

## 二、用户画像与使用场景

### 2.1 四类典型用户

| 用户类型 | 典型特征 | 核心诉求 |
|---------|---------|---------|
| **新手小白** | 第一次或几次滑雪，不知道怎么选雪场、准备什么、花多少钱 | "帮我规划一次滑雪出行" |
| **进阶玩家** | 滑过几个雪季，想找更适合自己的雪场、提升技术、关注性价比 | "推荐适合中级单板的雪场"、"帮我看看姿态" |
| **省钱达人** | 经常滑雪，关注早鸟票、季卡、转让票，愿意提前规划 | "什么时候买票最便宜"、"帮我盯着早鸟" |
| **旅行玩家** | 把滑雪和旅行结合，国内国外都去，关注完整体验 | "我想去日本/欧洲滑雪，帮我规划" |

## 三、四大场景详解

### 场景一：行程规划

**解决的问题**："我想去滑雪，但不知道去哪、怎么玩、花多少钱、天气怎么样"

**典型对话**：
- "周末想去崇礼滑雪，初学者，从北京出发"
- "推荐几个适合中级单板的雪场，预算 5000"
- "我想去日本滑雪，有什么推荐？"

**运行流程**：

```
用户输入
  │
  ▼
① 收集/确认画像（水平/城市/天数/预算）
  │
  ▼
② 推荐引擎匹配（resort_recommender.py）
    ├── 匹配用户水平（绿道/蓝道/黑道比例）
    ├── 匹配预算范围
    ├── 匹配偏好（粉雪/公园/陡坡/夜滑）
    └── 判断是否雪季 → 非雪季推荐室内
  │
  ▼
③ 生成攻略（按 travel-guide.md 模板）
    ├── 雪场对比（概况/雪道/票价/特色）
    ├── 天气查询（resort_recommender.py weather，未来 7 天预报）
    ├── 交通方案（高铁/自驾/飞机，含耗时和费用）
    ├── 住宿推荐（经济/中档/高端三档）
    ├── 装备清单（gear_guide.py，按水平/温度生成）
    ├── 保险建议（专项险推荐）
    └── 美食推荐（当地特色）
  │
  ▼
④ 自动查价（内部调用查价能力）
    ├── resort_recommender.py costs（数据库参考价）
    └── price_fetcher.py flyai-live（联网实时价）
  │
  ▼
⑤ 输出完整方案
    ├── 推荐 Top 3 雪场
    ├── 费用明细（交通+住宿+雪票+餐饮+装备）
    ├── 建议行程（Day 1 / Day 2 / ...）
    └── 实用贴士（安全/天气/APP）
```

### 场景二：智能查价

**解决的问题**："我已经定好去哪了，帮我查查要花多少钱，有没有便宜票"

**典型对话**：
- "帮我查下去北大湖滑雪要花多少钱，上海出发，1 月 15-18 号，2 个人"
- "万龙有没有便宜雪票，1 月 15 号去"
- "帮我看看闲鱼上有没有富龙的转让票"

**运行流程**：

```
用户输入
  │
  ▼
① 解析参数（雪场/出发地/日期/人数）
  │
  ▼
② 联网实时查价（首选）
    price_fetcher.py flyai-live
    ├── 查机票/火车票（实时航班价格）
    ├── 查酒店（携程/Booking 等实时报价）
    └── 查雪票（官网/OTA 价格）
  │
  ├─ 有结果 → 生成预算报告
  │
  └─ 无结果 → 降级
       ├── 数据库参考价（resorts_db.json）
       └── 标注"参考值（非实时）"
  │
  ▼
③ 同步搜索低价票（并行）
    WebSearch: "{雪场名} 残票 转让"
    WebSearch: "{雪场名} 特价票 {年份}"
    （闲鱼/飞猪/小红书/微信群）
  │
  ▼
④ 生成比价表（ticket_comparator.py）
    ├── 官方渠道价格
    ├── 二手转让价格
    └── 可信度标注
  │
  ▼
⑤ 输出
    ├── 实时预算明细（交通+住宿+雪票）
    ├── 低价票信息（如有，含来源和风险提示）
    └── 国际价格换算（currency_converter.py）
```

**输出示例**（简版）：

```markdown
## 北大湖滑雪预算（上海出发，1/15-1/18，2 人）

### 实时价格
| 项目 | 单价 | 2人合计 |
|------|------|---------|
| 机票（往返） | ¥1,280/人 | ¥2,560 |
| 酒店（3 晚） | ¥680/晚 | ¥2,040 |
| 雪票（3 天） | ¥480/人 | ¥960 |
| 餐饮 | ¥150/天/人 | ¥1,800 |
| **合计** | | **¥7,360**（人均 ¥3,680）|

### 低价票信息
暂未找到万龙转让票。建议：
- 关注万龙官方公众号，偶尔有促销
- 加入崇礼滑雪微信群，常有转票
- 购买多日票更划算（3 日票 vs 单×3）
```

### 场景三：AI 电子教练

**解决的问题**："帮我看看滑得怎么样，姿势对不对，有没有进步"

**典型对话**：
- "帮我看看这张万龙银龙道的照片，姿态怎么样"
- "帮我分析一下这个换刃视频"
- "我这个雪季滑得怎么样，有进步吗"

**运行流程**：

```
用户提交照片/视频
  │
  ▼
① 调用 ski_coach.py analyze
    ├── 输入：照片路径 + 雪场 + 雪道 + 难度
    └── 判断：是否配置了 API Key？
  │
  ├─ 有 API Key → 调用 Doubao 视觉 API 分析
  │
  └─ 无 API Key → 返回 agent_vision 模式
       └── Agent 用自身视觉能力分析
  │
  ▼
② 四维度评分（满分 10 分）
    ├── 基础姿态（重心、膝盖、上半身）
    ├── 转弯技术（平行站姿、立刃、换刃）
    ├── 自由式（仅 park/mogul 场景）
    └── 综合（速度控制、路线选择、地形适应）
  │
  ▼
③ 输出分析结果
    ├── 各维度评分
    ├── 亮点（做对的地方）
    ├── 问题（需要改进的地方）
    └── 建议（具体训练方法）
  │
  ▼
④ 自动记录（record → records.json）
    ├── 日期、雪场、雪道
    ├── 评分详情
    └── 照片引用
  │
  ▼
⑤ 长期价值（积累数据后）
    ├── progress → 进步趋势报告
    ├── season → 雪季总结回顾
    └── stats → 统计摘要（次数/雪场/均分）
```

**输出示例**（简版）：

```markdown
## 滑雪姿态分析 | 万龙滑雪场·银龙道（蓝道）

### 评分
| 维度 | 得分 | 说明 |
|------|------|------|
| 基础姿态 | 7.5/10 | 重心居中，膝盖弯曲良好，上半身略前倾 |
| 转弯技术 | 7.0/10 | 平行站姿到位，立刃角度可再加大 |
| 综合 | 7.2/10 | 路线选择合理，速度控制稳健 |

### 亮点 ✅
- 重心位置好，没有后坐
- 转弯时双板保持平行
- 视线方向正确，看着转弯方向

### 需要改进 ⚠️
- 上半身可再收一点，减少前倾
- 立刃角度偏小，建议加强脚踝灵活性
- 换刃过渡略慢，可在蓝道上练习节奏

### 训练建议
1. 平地练习：重心前后移动，找到中立位
2. 蓝道练习： focus 在立刃角度，尝试用脚踝控制
3. 录视频对比：每次练完拍一段，和上次对比
```

## 五、技术实现

### 5.1 脚本清单

| 脚本 | 用途 | 被谁调用 |
|------|------|---------|
| `resort_recommender.py` | 雪场推荐、画像管理、**高山天气查询**、费用估算、雪场对比 | 行程规划 |
| `price_fetcher.py` | 联网实时查价、搜索策略生成、预算报告 | 智能查价、行程规划 |
| `ski_coach.py` | 视觉分析、记录管理、进步报告、雪季总结 | AI 电子教练 |
| `presale_monitor.py` | 预售监听、状态检查、通知管理 | 早鸟预售 |
| `budget_calculator.py` | 手动预算计算（已知费用明细时） | 智能查价 |
| `ticket_comparator.py` | 多渠票价比价表 | 智能查价 |
| `gear_guide.py` | 按水平/目的地生成装备清单 | 行程规划 |
| `currency_converter.py` | 汇率换算 | 智能查价 |
| `utils.py` | 共享工具（数据加载、城市坐标、工具函数） | 全部 |

### 5.2 天气功能详解

行程规划模块自动包含**高山天气查询**，基于 Open-Meteo API 提供海拔高度定制化的天气预报：

```
resort_recommender.py weather '{"resort":"万龙滑雪场"}'
```

**输出内容**：
- 未来 7 天逐日预报（日期/天气/温度范围）
- **海拔温度**（非地面温度，基于雪场海拔高度计算）
- **体感温度**（考虑风寒效应，冬季体感通常比实际低 5-10°C）
- 降雪量（cm）、降雨量（mm）
- 最大风速 + 阵风（冬季大风对体感影响显著）
- **滑雪指数**（1-10 分，综合降雪/风速/温度评估适宜度）
- 最佳出行日推荐

**应用场景**：
- 出行前：选择最佳日期（新雪后 1-2 天粉雪最佳）
- 穿衣指导：基于体感温度推荐穿衣层数
- 安全提醒：大风/暴雪天气建议改期

### 5.3 降级策略

```
联网可用？
  ├── 是 → 获取实时数据
  └─ 否 → 数据库参考价
            │
            └─ 数据库也没有？ → 提示用户手动搜索
```

```

  ├── 有 → 调用 Doubao API 分析
```
